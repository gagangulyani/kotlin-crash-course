Functions

Cheat Sheet

Arrow function vibes with Python-like named params:

// Single expression
fun add(a: Int, b: Int): Int = a + b

// Default args + named params
fun greet(name: String, greeting: String = "Hi") = "$greeting, $name"

greet("Alex")                    // "Hi, Alex"
greet(name = "Alex", greeting = "Hey")  // named arguments

Collections

Cheat Sheet

Functional chaining like JS array methods:

val list = listOf(1, 2, 3)      // immutable

// Functional chaining
list.filter { it > 1 }
    .map { it * 2 }
    .forEach { println(it) }

Null Safety

Theory Essential

Kotlin's type system distinguishes nullable and non-nullable types at compile time. This eliminates NullPointerExceptions.

var name: String = "Alex"
name = null   // ❌ COMPILE ERROR

var nullable: String? = "Alex"
nullable = null  // ✅ OK, the `?` makes it nullable

// Safe call operator
val length = nullable?.length   // Int? (null if nullable was null)

// Elvis operator
val len = nullable?.length ?: 0  // default to 0 if null

// !! operator (avoid when possible)
val forced = nullable!!.length   // crashes if null

Key Insight: The ? in String? is not just syntax sugar—it's a fundamentally different type. The compiler enforces null checks, making impossible states unrepresentable.

Coroutines

Theory Essential

Lightweight threads managed by Kotlin (not OS). Code looks synchronous but executes asynchronously.

import kotlinx.coroutines.*

// Like async/await but structured
suspend fun fetchData(): String {
    delay(1000)  // non-blocking sleep
    return "Data"
}

fun main() = runBlocking {
    val job = launch {
        val data = fetchData()
        println(data)
    }
    job.join()
}

Key Insight: Think of suspend as async in Python/JS, but you call it directly without await inside other suspend functions. You can launch millions of coroutines—try that with threads!

Sealed Classes & When

Theory Essential

Restricted class hierarchies with exhaustive checking:

sealed class Result
data class Success(val data: String) : Result()
data class Error(val msg: String) : Result()

// Exhaustive when (like switch but smarter)
fun handle(result: Result) = when (result) {
    is Success -> println(result.data)
    is Error -> println(result.msg)
    // no else needed! Compiler knows all cases.
}

Key Insight: Perfect for state machines, API responses, and anywhere you need compile-time guarantees that you've handled all cases.

Jetpack Compose

Android

Declarative UI framework—like React but for native Android:

@Composable
fun Greeting(name: String) {
    Column(
        modifier = Modifier
            .fillMaxSize()
            .padding(16.dp),
        horizontalAlignment = Alignment.CenterHorizontally
    ) {
        Text(text = "Hello, $name!", fontSize = 24.sp)
        Button(onClick = { /* handle click */ }) {
            Text("Click me")
        }
    }
}

Concept: UI is a function of state. When state changes, the UI automatically recomposes.

ViewModel + StateFlow

Android

Architecture Components for reactive state management:

class MainViewModel : ViewModel() {
    // MutableStateFlow = mutable observable
    private val _count = MutableStateFlow(0)
    val count: StateFlow<Int> = _count.asStateFlow()

    fun increment() {
        _count.value++
    }
}

In Compose, collect as state:

val count by viewModel.count.collectAsState()
Text("Count: $count")

Kotlin Knowledge Check

Test your understanding before moving to Android!

Q1: What's the difference between val and var?

val is for numbers, var is for strings
val is immutable (like JS const), var is mutable (like JS let)
There is no difference; they're interchangeable

Q2: What does String? mean?

It's a String that must not be null
It's a nullable String that can hold null
It's an optional parameter type

Q3: What is the Elvis operator?

val len = name?.length ?: 0
Returns the left side if not null, otherwise the right side (default value)
Checks if both sides are equal
Throws an exception if the left side is null

Q4: What does suspend mean?

It pauses the entire app until the function completes
It can pause execution without blocking the thread, resuming later
It makes the function private

Q5: Why use data class?

It stores data in a local database automatically
Auto-generates equals(), hashCode(), toString(), copy(), and componentN()
It makes the class abstract
